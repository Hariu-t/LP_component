# システム要件定義書：コンポーネントベースLP制作プラットフォーム

## 1. プロジェクト概要

本システムは、再利用可能なUIコンポーネントを管理し、それらを組み合わせることで、**プログラミング知識のないユーザーでも高品質なLP（ランディングページ）を迅速に構築・公開できるDXツール**である。

## 2. システム構成（技術スタック）

モダンなフルスタック構成を採用し、パフォーマンスと運用効率を最大化する。

- **Frontend: Next.js (App Router)**
  - 管理画面および公開用LPの高速レンダリング（ISR/SSG）を担当。

- **Backend & DB: Supabase**
  - PostgreSQLによるデータ管理
  - Authによる認証
  - Storageによる画像保存

- **Infrastructure: Vercel**
  - エッジ配信によるグローバルな高速表示
  - デプロイ自動化

- **Styling: Tailwind CSS**
  - コンポーネント間のスタイル干渉を防止
  - 自由度の高いデザインを実現

## 3. 機能要件

### 3-1. コンポーネント管理機能

エンジニアが「部品」を登録し、マーケターが「再利用」するための基盤。

- **コンポーネントCRUD**
  - HTML/CSSテンプレートの登録・編集・削除
- **Props（変数）定義**
  - 各部品で編集可能な項目（テキスト、画像、リンク、背景色等）をJSONB形式で定義
- **使用状況トラッキング**
  - 各コンポーネントがどのページで使用されているかをカウントし、影響範囲を可視化

### 3-2. ページ制作（エディタ）機能

Studioのような直感的な操作を実現するコア機能。

- **サイトマップ管理**
  - ページ階層（ディレクトリ構造）の作成・管理
- **ドラッグ＆ドロップ編集**
  - 登録済みコンポーネントをリストから選び、順序を入れ替え
- **リアルタイムインスペクタ**
  - 配置したコンポーネントをクリックし、右パネルで内容（Props）を書き換え
- **デバイスプレビュー**
  - PC / タブレット / スマホの表示切り替え

### 3-3. 公開・配信機能

- **ワークフロー管理**
  - 「下書き保存」「プレビュー」「公開」のステータス管理
- **オンデマンドISR**
  - ページ公開時にVercelのキャッシュをパージし、即座に反映
- **独自ドメイン設定**
  - 公開したLPを特定のドメインに紐付け

## 4. データモデル設計（主要テーブル）

| テーブル名      | カラム例（主要なもの）                            | 説明                                                         |
|-----------------|---------------------------------------------------|--------------------------------------------------------------|
| `components`    | `id`, `name`, `html_template`, `schema_json`      | 部品の定義と編集可能項目の設計図。                          |
| `pages`         | `id`, `slug`, `title`, `status`, `parent_id`      | LPの基本情報と階層構造。                                    |
| `page_sections` | `id`, `page_id`, `component_id`, `content_json`, `order` | ページ内の「どの位置」に「どの部品」を「どんな内容」で置くか。 |
| `assets`        | `id`, `url`, `file_type`, `alt_text`              | 画像・動画ファイルの情報。                                  |

## 5. 非機能要件

- **パフォーマンス**
  - 公開ページはLighthouseのパフォーマンススコア90以上を目標とする。
- **セキュリティ**
  - SupabaseのRLS (Row Level Security) を活用し、テナント間やユーザー間でのデータ漏洩を防止。
- **拡張性**
  - 将来的に外部CRM（HubSpot等）とのAPI連携が可能なプラグイン構造を考慮する。

## 6. 画面設計イメージ

- **コンポーネント一覧画面**
  - カード形式で各部品を表示。
  - 「使用中：5ページ」といったバッジを表示。
- **エディタ画面**
  - 左にツリー（階層）
  - 中央にライブプレビュー
  - 右に編集パネル（Property Editor）を配置。
